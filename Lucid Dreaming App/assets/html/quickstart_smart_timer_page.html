SmartTimer configuration

Smart Timer is a self adjusting timer, it checks user gestures and movement to adjust itself
To help you with configuration, I added 2 Smart Timer files - the one that I use and a blank one. Just use one of them as a template
Smart timer uses times after going to bed, so event afte 2:00 will take place 2 hours after going to bed. If you go to bed at 1AM, it will take place at 3AM, etc
  • The built-in timer is properly configured and requires minor adjustments
  • The timer has a set of REM events - when you are likely to be dreaming. A reminder will be played during that time interval.
  • Add or edit the REM events present in SmartTimerConfig.txt.gzip by clicking the change config button below
  • Assign reminders to REM events. They show as colored Xs on the graph below
  • Deactivate events when you would like no reminders to be played.
To adjust the sensitivity of Smart Timer to movement, click the Smart Timer Preferences button and change the REM Prediction activity threshold. Lower values cause the reminder to be played closer to the REM event start, while higher values may never get played. Below is the chart of the currently selected Smart Timer config
  • Green lines indicate active events - these will play reminders. Gray lines are not active and will not play anything
  • Activate/deactivate events rather than delete them
  • An X is shown above an event if a reminder is properly configured for that event
  • Reminders above beginning/end of a line will be played at start/end
  • Reminders above the middle of the line will be played on movement
Please note that Xs at 0 are for display purposes only and the graph below updates only when config is saved!
